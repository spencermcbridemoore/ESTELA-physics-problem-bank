# ESTELA lean text-only MCQ export

Structured extract of the **95 text-only, single-answer multiple-choice** questions
from the ESTELA physics problem bank — the subset best suited to MCQ logprob work.
Generated by `build_lean_mcq.py` (re-runnable; reads the repo, writes only here).

## Files
| file | what |
|---|---|
| `estela_text_only_mcq.jsonl` | one JSON object per question — primary format for a logprob/eval harness |
| `estela_text_only_mcq.csv` | flattened, spreadsheet-friendly; **choice columns are in the de-biased shuffled order** |
| `build_lean_mcq.py` | the extractor; edit `LEAN_BANKS` to add/remove banks |

## JSONL field dictionary
- `uid` — `<bank_id>::<qid>`, stable key.
- `bank_id`, `bank_title`, `unit`, `learning_objectives`, `source_file`, `source_qid` — provenance.
- `title`, `points`, `has_figure` (always `false` here), `num_choices`, `labels`.
- `question` — cleaned: `<latex>…</latex>` → `$…$`, HTML stripped. `question_raw` — original.
- `choices` / `choices_raw` — cleaned / original option text, **source order**.
- `answer_index` / `answer_label` / `answer_text` — correct option in **source order**.
- `shuffle_perm`, `choices_shuffled`, `answer_index_shuffled`, `answer_label_shuffled`
  — **de-biased order** (deterministic, seeded from `uid`; reproducible). `shuffle_perm[new] = old_index`.
- `format_flags` — per-question quirks (see below).

## Caveats that matter for logprob MCQ
1. **Strong position bias.** 70/95 correct answers are at position **A** in the source
   (three banks are 100% A). Use the `*_shuffled` fields (JSONL) or the CSV, which is
   already shuffled — otherwise "always A" scores 74% and position confounds everything.
2. **Duplicate distractors** on 3 questions (`format_flags: duplicate_choices`):
   `PHY1-SMH-MESHM…::q-1`, `::q-2`, `PHY1-F-ACCAC…::q18` — two option strings are identical,
   so probability mass splits across indistinguishable labels. Consider dropping or dedup.
3. **Variable option counts.** SMH-MESHM uses 8–10 options; the other four use 4–5.
   Normalize per-question if your metric assumes a fixed option set.
4. **Answer text is content, not a letter.** Options are LaTeX expressions or values with
   units (e.g. `503.34 N`). To read a model's logprob over a *choice*, inject your own
   `A/B/C/D` labels — don't expect a bare letter in the data.
5. **`bank_id` ≠ folder** for one bank: folder `PHY1-N-DACCFMC-10082025` declares
   `bank_id: PHY1-F-ACCAC-10082025`. `uid` follows the declared id.
6. **One source file has malformed YAML** (`PHY1-F-CWDVF-…`: `status:` is over-indented under
   `bank_info`), which strict parsers reject even though it's marked `status: ready`. The
   extractor sidesteps it by parsing the `questions:` block in isolation.

## Not included
`PHY1-VEC-VASTV-09072025` (10 MC) has a `figure:` field but is fully text-solvable (all
magnitudes/angles are in the prompt). To add it, append its path to `LEAN_BANKS` and re-run;
it will carry `has_figure: true` so you can filter later.
